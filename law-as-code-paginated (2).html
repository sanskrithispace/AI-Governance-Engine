Law-as-Code
Age Verification System

A proof-of-concept showing how UAE child digital safety laws (Federal Decree-Law No. 26 of 2025) can be made machine-readable, auto-enforced, and updated without any code changes.

⚖ Basis: UAE Federal Decree-Law No. 26 of 2025
Prohibits collection/processing of child data under 13 without parental consent · Mandates risk-based age verification · Platforms must block gambling, profiling, and targeted ads for minors · Generative AI banned for under-13s or below Year 7.

⚙ How the Engine Works
① Identity Gate
Validate Emirates ID format → run face scan → check confidence score ≥ 80%.
Fail → BLOCK immediately. No laws are checked. This prevents spoofing.
② Read laws.json (Layer 3)
Engine loads ALL active laws from the data layer. No rules hardcoded here.
Add a new law to the registry → engine picks it up on the next verification, automatically.
③ Evaluate All Rules Simultaneously
Every active law runs against (age + service_type). Each returns PASS / FAIL / N/A.
Nothing is skipped. This prevents gaps when multiple laws apply.
④ Combined Final Decision
Priority: BLOCK > REQUIRE_CONSENT > ALLOW
The strictest triggered rule wins. Results in one definitive decision stored to Layer 3.
⑤ Auto-Expiry + Store Record
If user was previously blocked but birthday has now passed the threshold → block is lifted automatically.
Final decision hashed with SHA-256 and appended to the immutable audit log (Layer 3).
